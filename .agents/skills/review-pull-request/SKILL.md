## 処理内容

- issue番号をもとに、issueの内容を読み取る
  issue番号はブランチ名から取得する。ブランチ名の形式は、`*/issue_#<issue番号>` とする。
- ADR(Architectural Decision Record)を読み取る。
  格納ディレクトリ: `docs/adr/<issue番号>_*.md` ※issue番号はゼロパディングありの数字
  ADRが存在しない場合は、ADRを読み取らず、issueの内容だけでレビューを行う。
- issueおよびADRの内容をもとに、要件として整理する。
- 要件に対して、実装状況を確認する。実装状況は、ソースコードの修正内容、テスト結果、残課題などをもとに判断する。
- 実装状況に基づき、指摘事項を整理する。指摘事項は、重要度順で並べ、各項目にファイル名と行番号を含める。
- 実装状況に基づき、確認事項を整理する。確認事項は、実装状況の不備や、テスト結果の不備、実装上の不具合などが含まれる。
- 実装状況に基づき、概要を整理する。概要は、実装状況の全体像を簡潔にまとめる。
- 実装状況に基づき、結果(問題なし、要修正、コメントあり)を判断する。
- **必須出力形式** に従って、レビュー内容をまとめる。

## 必須出力形式
次の見出しで結果を出力する。

```
## 結果
## 概要
## 要件と実装状況
## 指摘事項
重要度順で並べ、各項目にファイル名と行番号を含める。
## 確認事項
```
